# 📁 项目文件说明

```
fishing-mcp-server/
├── engine.py              # 🎮 游戏引擎核心（包含所有游戏逻辑）
├── server.py              # 🔌 MCP 服务器（stdio 模式，本地连接用）
├── app.py                 # 🌐 HTTP 服务器（Render 部署用）
├── requirements.txt       # 📦 Python 依赖
├── .python-version        # 🐍 Python 版本指定
├── .gitignore             # 🚫 Git 忽略文件
├── README.md              # 📖 项目说明
├── 部署指南.md            # 🚀 详细部署教程
├── test_local.py          # 🧪 本地测试脚本
└── fishing_save.json      # 💾 游戏存档（自动生成，不提交到 Git）
```

## 各文件作用

### 核心文件

**engine.py** - 游戏引擎
- 完整的钓鱼游戏逻辑
- 包含鱼类、地点、鱼饵数据
- 确定性随机数生成器
- 存档系统
- 对外提供 `cmd(command)` 和 `new_game(seed)` 接口

**server.py** - 本地 MCP 服务器
- 实现 MCP 协议（stdio 传输）
- 用于本地连接 Claude Code
- 读取 stdin，输出到 stdout

**app.py** - 远程 HTTP 服务器
- Flask Web 应用
- 提供 `/health` 健康检查接口
- 提供 `/mcp` MCP 协议接口
- 用于 Render 等云平台部署

### 配置文件

**requirements.txt** - 依赖管理
- Flask==3.1.0
- Werkzeug==3.0.6

**.python-version** - Python 版本
- 指定使用 Python 3.11.0

**.gitignore** - Git 忽略规则
- 存档文件（fishing_save.json）
- Python 缓存（__pycache__, *.pyc）
- 虚拟环境（venv/）

### 文档

**README.md** - 项目说明
- 功能特性
- 本地测试方法
- 部署步骤概述
- 可用工具列表

**部署指南.md** - 详细教程
- 完整的部署步骤（小白友好）
- GitHub 上传教程
- Render 配置指南
- 常见问题解答

**test_local.py** - 测试脚本
- 自动测试所有游戏功能
- 验证游戏逻辑正确性
- 测试完自动清理存档

## 使用场景

### 场景 1: 本地开发测试
```bash
python test_local.py          # 运行测试
python engine.py              # 直接玩游戏
```

### 场景 2: 本地连接 Claude Code
```bash
python server.py              # 启动 stdio MCP 服务器
# 然后在 Claude Code 配置中添加此服务器
```

### 场景 3: 部署到 Render
```bash
# Render 会自动运行
python app.py                 # 启动 HTTP 服务器（端口 10000）
```

## 下一步开发

想要扩展功能？修改这些文件：

1. **添加更多鱼/地点** → 编辑 `engine.py` 顶部的数据字典
2. **添加新功能** → 在 `engine.py` 的 `cmd()` 方法中添加新指令
3. **修改 MCP 工具** → 编辑 `server.py` 或 `app.py` 的工具定义

修改后别忘了测试：
```bash
python test_local.py
```
